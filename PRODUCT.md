# PRODUCT.md — data-nexus

> impeccable 上下文锚点。描述"产品是什么、给谁用、什么调性"，所有前端设计据此判断。

## 一句话
轻量级内部**数据共享门户**：管理员把 SQL 跑成 Excel、归入文件夹并按部门/个人授权；用户登录后浏览有权限的文件夹、下载文件。

## register
**product**（应用 UI / 后台 / 工具）。设计 SERVES 任务，不是品牌橱窗。目标是"earned familiarity"——像 Linear / Notion / Google Drive 那样让人一坐下就信任、工具消失在任务里。不是"看起来很 AI"，而是"每个组件都恰当、不别扭"。

## 用户与场景
- **管理员 / 辅助管理员**：配数据源、写数据集（SQL+定时+命名+保留）、建文件夹目录、配权限（人/部门→可见文件夹）、管部门与人员。任务密集，要效率与清晰。
- **普通用户（老板/总监/员工）**：进门户，左侧看自己有权限的文件夹树，右侧浏览/搜索/下载文件。低频、轻量、要"友好、一眼会用"。

## 调性 / 北极星
- **浅色系**（一套冷调中性 + 单一靛蓝强调色，克制；强调色只用于主操作/当前选中/状态，不做装饰）。
- 安静、专业、密而透气。**动效服务状态**（hover/选中/加载/进入），150–250ms，不做开场编排动画。
- 骨架屏而非转圈；空状态教用户怎么用；每个交互组件都有 hover/focus/active/disabled/loading 态。

## 不要（slop）
渐变文字、侧边色条、默认玻璃拟态、千篇一律卡片网格、每段小写 eyebrow、装饰性动效、给标准操作发明奇怪控件、低对比灰字正文。

## 技术约束
Vue 3 + Element Plus + Vite（不换栈）。用 skill 的设计原则改"皮"与交互，保留 Element 的表格/弹窗/表单能力。
