# changelog（系统变更说明）

本目录记录每个版本的变更，**每次向 GitHub 推送新版本时必须新增/更新一份**。

## 约定

- 每个版本一个文件，命名：`vX.YZ.md`（如 `v0.01.md`）。
- **版本号从 `v0.01` 起，每完成一个版本 +0.01 依次累计，不跨版本**；重大里程碑进位大版本（如 `v1.00`）。
- 每份文档包含两部分：
  1. **本次变更**：本版本新增、修改、修复、删除了什么（删除项必须列明，呼应开发规范第 2 节）。
  2. **下一版本计划**：下一个版本打算做什么。
- 版本号、提交记录、本文档、`system-docs/` 四者保持一致。

## 模板

见 [template.md](template.md)，新建版本时复制改名即可。
