# v0.03 版本变更说明

> 发布日期：2026-06-04　|　对应分支/标签：main　|　负责人：henry + AI（技术总监）
> 主题：**前端项目骨架 + 登录认证打通**

## 一、本次变更

### 新增（feat）
- 初始化前端 `frontend/`（Vite + Vue 3 + TypeScript）：
  - 集成 Element Plus、Pinia、vue-router、axios。
  - `vite.config.ts`：`@` 别名 + `/api` 代理到后端（开发免跨域）。
  - `src/api/http.ts`：axios 实例，注入 Token、401 跳登录、统一错误提示。
  - `src/stores/auth.ts`：Pinia 登录态（Token 持久化到 localStorage）。
  - `src/router/index.ts`：路由 + 登录守卫，视图懒加载。
  - `src/layouts/MainLayout.vue`：侧边菜单 + 顶栏 + 退出。
  - `src/views/`：登录页、概览页（拉 `/auth/me/` 验证连通）、数据源/任务/执行记录占位页。
  - ESLint（扁平配置，Vue3+TS 官方）+ Prettier + `.env.development`。
- 后端新增认证端点（登录是全栈功能）：
  - `POST /api/auth/token/`（账号密码换 Token）、`GET /api/auth/me/`（当前用户）。

### 修改 / 优化
- 覆盖 Vite 默认 `style.css` 为最小 reset，避免默认样式限制布局宽度。
- `health` 接口版本号 v0.02 → v0.03。

### 删除
- 删除脚手架自带的未使用组件 `src/components/HelloWorld.vue`（已被自有 `App.vue` 取代，无引用）。**已告知**。

## 二、验证（三维度审核）

- **维度一·功能正确性**：前端 `npm run build`（含 vue-tsc 类型检查）通过、`eslint` 通过；后端登录链路冒烟：登录 200→带 Token 取 me 200（admin）→无 Token 401。
- **维度二·健壮性/扩展性**：401 自动跳登录 + 统一错误提示；路由守卫；Token 持久化；视图懒加载；布局/菜单可扩展。
- **维度三·关联影响**：后端仅新增 URL 路由（附加式），回归 `ruff`/`black`/`pytest` 24 例全过，既有安全核心未受影响。
