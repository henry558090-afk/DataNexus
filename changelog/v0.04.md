# v0.04 版本变更说明

> 发布日期：2026-06-05　|　对应分支/标签：main　|　负责人：henry + AI（技术总监）
> 主题：**前端端口调整为 5179 + 系统本地启动验证**

## 一、本次变更

### 修改 / 优化
- 前端开发端口 `5173` → **`5179`**（`frontend/vite.config.ts`）。
- 后端 `CORS_ALLOWED_ORIGINS` 同步改为 5179（`.env.example` 及本地 `.env`）。
- `health` 接口版本号 v0.03 → **v0.04**。
- 同步更新 `system-docs/architecture.md` 前端启动端口说明。

### 修复
- 修复本地 `.env` 因 PowerShell 5.1 `Set-Content -Encoding utf8` 写入 BOM + 中文乱码、导致 django-environ 报 "Invalid line" 的问题：改用 Python 以 UTF-8（无 BOM）重新生成 `.env`。

### 新增 / 删除
- 无。

## 二、验证（三维度审核）

- **维度一·功能正确性**：本地启动后端(8000)+前端(5179)，逐项验证：
  - 后端健康 `GET /api/health/` → `{"version":"v0.04"}`；
  - 前端代理 `http://localhost:5179/api/health/` → 正常（证明 Vite proxy 生效）；
  - 登录 `POST /api/auth/token/`（admin）→ 返回 Token；
  - 前端页面 `http://localhost:5179/` → HTTP 200。
- **维度二·健壮性/扩展性**：仅配置调整，无逻辑变更。
- **维度三·关联影响**：端口为开发期配置，不影响后端业务与既有功能；后端接口验证正常。

## 三、使用说明

- 访问地址：**http://localhost:5179**（Vite 绑定 IPv6 `localhost`，勿用 127.0.0.1:5179）。
- 本地账号：`admin / admin12345`（存于 gitignore 的 db.sqlite3，不进仓库）。
- 启动命令见 `system-docs/architecture.md` 第 5.1 / 5.2 节。
