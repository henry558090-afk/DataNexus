# v0.07 版本变更说明

> 发布日期：2026-06-05　|　分支：**dev**（重构基线大改版 · 阶段 A 地基）　|　负责人：henry + AI
> 主题：**前端重构 —— 双布局（用户端/管理端）+ 角色路由分流**

## 一、本次变更

### 新增（feat）
- **双布局**：
  - `layouts/AdminLayout.vue`：管理端（侧边菜单工作台：工作台/数据源/数据集/目录管理/用户与权限/执行记录 + 切换到用户端）。
  - `layouts/UserLayout.vue`：用户端数据门户（商务风顶栏 + 头像下拉，管理员可"进入管理端"）。
- **角色分流**：`router` 拆 `/admin`（`requiresManager` 守卫）与 `/`（用户端）两棵路由树；登录后超管/辅助管理员 → `/admin`，普通用户 → `/`。
- 管理端页面（占位，统一 `PageContainer`）：`AdminHome` + 数据源/数据集/目录/权限/执行。
- 用户端首页 `user/CatalogHome.vue`：门户欢迎 + 搜索 + 部门浏览占位。
- 后端 `/api/auth/me/` 扩展返回角色：`is_superuser / is_assistant_admin / is_boss / is_manager`。

### 修改 / 优化
- `stores/auth.ts`：登录后拉取 `profile`，新增 `isManager`；`LoginView` 按角色跳转。
- `health` 版本号 → v0.07。

### 删除（已告知）
- `layouts/MainLayout.vue`、`views/DashboardView.vue`、`views/DataSourceView.vue`、`views/TaskView.vue`、`views/ExecutionView.vue`：被双布局结构（`AdminLayout/UserLayout` + `views/admin/*`、`views/user/*`）取代，功能上移/重排，非能力删除。

## 二、验证（三维度审核）

- **维度一·功能正确性**：前端 `build`（vue-tsc）+ `eslint` 通过；`/api/auth/me/` 冒烟返回角色（admin → `is_manager=true`，分流至 `/admin`）；后端 `check` 0 问题、35 测试全过。
- **维度二·健壮性/扩展性**：守卫在缺 profile 时自动拉取、401 交拦截器；双布局复用浅色令牌；占位页统一容器，后续填充不动框架。
- **维度三·关联影响**：删除的旧文件均有替代；后端 me 改动为附加式，35 测试不受影响。
