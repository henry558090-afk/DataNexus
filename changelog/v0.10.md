# v0.10 版本变更说明

> 发布日期：2026-06-06　|　分支：**dev**（重构基线大改版 · 质量加固）　|　负责人：henry + AI
> 主题：**全面代码审查 + 缺陷修复 + 最优化加固**

本版不加新功能，专门对 v0.06–v0.09 已开发功能做一次彻底自审，找出并修复真实缺陷，提升到更优实现。

## 一、审查发现与修复

### 🔴 A（BUG）无分类数据集导致可见性判定崩溃
- **问题**：v0.09 将 `Dataset.category` 改为可空后，`can_view_dataset` 中 `dataset.category.department_id` 在 `category=None` 时抛 `AttributeError`。该函数是未来用户端目录/下载的唯一权限出口，属潜伏严重 BUG。
- **修复**：增加 `category_id is None` 守卫——未归类数据集对非管理员一律不可见（管理员/老板已在前置放行）。
- **回归测试**：`test_null_category_denied_for_non_admin` / `test_null_category_visible_for_admin`。

### 🟠 B（缺陷）0 行结果丢失列名 → 导出 Excel 无表头
- **问题**：`oracle_client._execute_stream` 在结果为空时直接 break，不交付列名；`stream_query` 因而返回空列名，导出的 Excel **连表头都没有**。
- **修复**：空结果也 `yield columns, []`，保证表头始终写入。
- **回归测试**：`test_export_empty_rows_writes_header_only`（导出空数据仅有表头）。

### 🟡 C（性能）数据集列表 N+1 查询
- **问题**：`DatasetSerializer.get_latest` 对每个数据集单独查一次最新执行，列表场景 N+1。
- **修复**：ViewSet 用 `Prefetch(is_latest=True, to_attr="latest_execs")` 预取；序列化优先用预取结果，详情/创建回退查询。

### 🟢 D（健壮性）运行落库非原子
- **问题**：`run_dataset` 成功路径"清旧 is_latest + 保存新执行"非原子，中途崩溃可能留下不一致。
- **修复**：包入 `transaction.atomic()`。

## 二、验证（三维度审核）

- **维度一·功能正确性**：`check` 0；`ruff`/`black` 全清；`pytest` **53 例全过**（50 + 3 回归）。
- **维度二·健壮性/扩展性**：空守卫、表头保障、预取优化、事务原子，均为更优实现。
- **维度三·关联影响**：修改后既有 50 测试仍全过，无回归。

## 三、已知限制（**有意延后**，附理由，非缺陷）

技术总监如实标注，避免"假装完美"：
1. **列表接口未分页**：执行记录会随时间增长。当前内部规模数据量小，暂不分页；待数据量上来再加 DRF 分页（会同步调整前端）。
2. **运行为同步**：手动运行大查询会阻塞该请求。适合中小数据；大数据/定时的后台执行随 **v0.12 调度** 引入。
3. **预览 CLOB/LOB 列**：oracledb 的 LOB 对象 JSON 序列化需特殊处理，待接真实 Oracle 时按需加（转字符串/截断）。
