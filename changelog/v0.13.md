# v0.13 版本变更说明

> 发布日期：2026-06-06　|　分支：**dev**　|　负责人：henry + AI
> 主题：**MySQL 支持 + 管理端首页重做 + 用户端门户重构**

## 一、本次变更

### 新增（feat）
- **多数据库支持（Oracle + MySQL）**：
  - 新增 `core/db.py`：统一取数客户端，按 `db_type` 连 Oracle（oracledb thin）或 MySQL（PyMySQL）；DB-API 通用执行逻辑，MySQL 流式用 `SSCursor`；只读校验/绑定变量/行数上限/超时不变。
  - `DataSource` 新增 `db_type` 字段（oracle/mysql，迁移 0002）；序列化暴露；测试连接/预览/运行全部按类型分发。
  - 依赖新增 `PyMySQL`。
- **管理端首页统计接口** `GET /api/stats/`（数据源/数据集/执行/今日运行计数）。
- **前端**：
  - 数据源表单新增「类型」单选（Oracle/MySQL）+ 切换自动套默认端口（1521/3306）+ MySQL 时"服务名"改"数据库"；列表显示类型；数据集选数据源时标注库类型。
  - **管理端首页重做**：真实统计卡片（可点击跳转）+ **最近运行**列表，**移除无意义的"快速开始"步骤条**。
  - **用户端门户重构**：左侧**部门/分类树** + 顶部**搜索** + 数据卡片列表——文件多也能快速定位（按部门/分类筛选或全局搜索）。

### 修改 / 优化
- `services`/`datasource.views` 改用 `core.db`；`health` → v0.13。

### 删除（已告知）
- 删除 `core/oracle_client.py`：被通用的 `core/db.py` 取代（功能上移并扩展为双库），相关测试 mock 同步迁移到 `core.db`。
