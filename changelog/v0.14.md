# v0.14 版本变更说明

> 发布日期：2026-06-06　|　分支：**dev**　|　负责人：henry + AI
> 主题：**定时调度（自动跑 SQL）+ 执行历史保留清理**

## 一、本次变更

### 新增（feat）
- **定时调度（APScheduler，进程内）**：
  - `core/scheduler.py`：`build_trigger`（cron 优先，其次 interval）+ `reconcile`（把数据集定时配置与调度任务对齐，增/删/改自动生效）。
  - 管理命令 `python manage.py runscheduler`（BlockingScheduler，每 30s reconcile），须单进程运行（技术方案 §10.1）。
  - 依赖新增 `APScheduler`。
- **执行历史保留**：`settings.EXECUTION_KEEP`（默认 20）；每次运行后保留最近 N 次，超出连同 Excel 文件自动清理（`_prune_executions`）。
- **前端**：数据集表单新增「定时」配置（手动 / 间隔分钟 / 每天 HH:MM / Cron 表达式，互转 cron），数据集列表显示定时摘要。

### 修改 / 优化
- `health` → v0.14。

### 删除
- 无。

## 三、使用

- 启动调度：`cd backend && .\.venv\Scripts\python.exe manage.py runscheduler`（与 web 分开的进程）。
- 在数据集编辑里选「每天 08:00」即可每天自动跑出 Excel。
