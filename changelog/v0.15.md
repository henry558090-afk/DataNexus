# v0.15 版本变更说明

> 发布日期：2026-06-06　|　分支：**dev**　|　负责人：henry + AI
> 主题：**审计日志 + 列表分页 + 生产安全加固**（朝生产可用收口）

## 一、本次变更

### 新增（feat）
- **审计日志**：新增 `apps/audit`（`AuditLog` 模型 + `log()` 助手，审计失败不阻断业务）。
  - 留痕动作：**登录**（自定义 `LoginView` 包装 obtain_auth_token）、**运行数据集**、**下载文件**（管理端下载 + 用户端门户下载）。
  - `GET /api/audit-logs/`（仅管理员，分页，?action/?user 过滤）。记录账号/动作/对象/IP/时间。
- **列表分页**：新增 `core/pagination.py`；对**增长型**列表 `/api/executions/`、`/api/audit-logs/` 启用分页（`{count,next,previous,results}`，page_size=20）；其余有界列表保持数组。
- **生产安全加固**（settings）：`SECURE_CONTENT_TYPE_NOSNIFF`、`X_FRAME_OPTIONS=DENY`；**非 DEBUG** 时启用 `SESSION/CSRF_COOKIE_SECURE`、`SECURE_PROXY_SSL_HEADER`、HSTS。
- **前端**：管理端新增**审计日志页**（筛选/分页）+ 菜单；执行记录页改**分页**；首页"最近运行"适配分页结果。

### 修改 / 优化
- 登录端点改为 `LoginView`（保持返回 `{token}`，附带审计）；`health` → v0.15。

### 删除
- 无（移除 urls 中未用的 obtain_auth_token 导入）。

## 二、验证（三维度审核）

- **维度一·功能正确性**：`pytest` **79 例全过**（+5：登录/运行/下载留痕、审计分页过滤、权限）；`ruff`/`black` 全清；前端 `build`/`eslint` 通过。**实时冒烟**（8010）：health v0.15、登录留痕、审计分页（IP 记录正确）。
- **维度二·健壮性/扩展性**：审计写入异常不阻断主流程；分页只加在增长型列表（避免破坏既有数组消费）；安全头仅生产启用，不影响本地。
- **维度三·关联影响**：登录端点替换保持响应结构不变；前端执行列表两处消费同步改为 `.results`；既有 74 测试全过。

## 三、生产就绪状态

到 v0.15，平台具备：双库取数 → 定时/手动跑 → Excel → 按部门编目 → 角色权限分发 → 受控下载 → **全程审计** → 历史保留清理 → 列表分页 → 基础安全加固。
