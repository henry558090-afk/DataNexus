# v0.16 版本变更说明

> 发布日期：2026-06-07　|　分支：**dev**　|　负责人：henry + AI
> 主题：**最小生产化 —— 单服务发布（无 Nginx / Docker / PostgreSQL）**

## 一、本次变更

### 新增（feat）
- **单服务发布**：引入 **WhiteNoise**，Django（gunicorn）一个进程同时发**前端 SPA**（`frontend/dist`）与 `/api`，**不需要 Nginx**。
  - `config/urls.py` 新增 SPA catch-all（非 api/django-admin/static/media 路径返回 `index.html`，支持前端 history 路由刷新）。
  - **Django 后台从 `/admin/` 挪到 `/django-admin/`**，把 `/admin/*` 让给前端管理端路由（修复生产路由冲突）。
- **元数据库可切换**：默认 **SQLite（开 WAL + busy_timeout + timeout=20）**，缓解 web/调度双进程并发写；可用 `DATABASE_URL` 切到现有 **MySQL**（`accounts.apps.ready` 设 SQLite PRAGMA）。
- **生产依赖与配置**：`requirements-prod.txt`（gunicorn）；`whitenoise` 入基础依赖；日志（非 DEBUG 滚动写 `logs/app.log`）；`.env.example` 补 `DATABASE_URL`/`EXECUTION_KEEP`。
- **部署文档** `docs/部署.md`：Linux 最小部署（venv + build + 两个进程 + systemd 守护 + 升级/备份），明确不需要 Docker/Nginx/PG。

### 修改 / 优化
- `health` → v0.16；`.gitignore` 忽略 `logs/`、SQLite WAL 临时文件。

### 删除
- 无。

## 二、验证（三维度审核）

- **维度一·功能正确性**：`pytest` **79 例全过**；`check`/`ruff`/`black` 全清；前端 `build` 通过。**单服务冒烟**（test client）：`GET /` → 200 发 index.html；`GET /admin/datasets`（前端路由）→ 200；`GET /api/health/` → v0.16；`GET /django-admin/` → 302（后台正常）。
- **维度二·健壮性/扩展性**：SQLite WAL 缓解并发写、可平滑切 MySQL；单服务降低运维面；日志可排查。
- **维度三·关联影响**：后台路径迁移仅影响 `/admin/`（前端未用该 Django 路径）；WhiteNoise/DB 改动经 79 测试全过；新增 catch-all 排除 api/django-admin/static/media，不影响接口。

## 三、上线只需

1. `pip install -r requirements-prod.txt`，`npm run build`。
2. 填 `.env`（`SECRET_KEY`/`DEBUG=False`/`ALLOWED_HOSTS`/**`FERNET_KEY` 并备份**）。
3. `migrate` + `collectstatic` + `createsuperuser`。
4. 起两个进程：`gunicorn config.wsgi`（:8000）+ `manage.py runscheduler`（systemd 守护）。
5. 提供真实 Oracle/MySQL 只读账号做端到端真数据验证。
