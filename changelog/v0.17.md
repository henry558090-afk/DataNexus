# v0.17 版本变更说明

> 发布日期：2026-06-07　|　分支：**dev**　|　负责人：henry + AI
> 主题：**核心重构（后端）——从"数据集+分类"转为"文件夹 + 文件 + 文件夹授权部门/个人"的网盘式模型**

> ⚠️ 本版仅后端模型/接口重构。**前端将在 v0.18（管理端）/ v0.19（用户端文件浏览器）随后重做**；在那之前前端功能与新接口不匹配，属重构中间态（仅 dev 分支）。

## 一、数据模型重构

| 旧 | 新 |
|---|---|
| 部门 → 分类(Category) → 数据集 → 执行版本(Execution，保留 N) | **部门(用户组) ／ 文件夹(Folder 多层树) ／ 数据文件(DataFile)** |
| Grant（个人/角色组 → 分类/数据集） | **FolderShare：文件夹 → 部门 或 个人**（递归覆盖子文件夹/文件） |
| 部门内 4 级角色（老板/总监/主管/成员） | **删除细分**；仅"部门归属(可多部门)" + 全局 老板/超管/辅助管理员 |
| 数据集每次跑覆盖"最新版本" | 数据集每次跑**在目标文件夹新增**一个**带日期命名**的文件（`前缀_YYYYMMDD.xlsx`），按 `keep_count/keep_days` 保留清理 |

- 新增：`catalog.Folder`(树, 防环移动) / `catalog.FolderShare` / `execution.DataFile`；`dataset.Dataset` 加 `target_folder/file_prefix/date_format/keep_count/keep_days`，去 `category`。
- 删除：`catalog.Category` / `permission.Grant` / `execution.Execution` / 部门成员的 role/see_all。
- **可见性唯一出口重写**：`can_view_folder/can_view_file`（祖先文件夹被授权给"我的部门或本人"即可见，默认拒绝）+ `visible_folders`（授权文件夹及其子孙）。

## 二、接口变化

- 新增/改：`/api/folders/`（CRUD + 重命名 + 移动(改 parent, 防环) + 删）、`/api/folder-shares/`（文件夹→部门/个人）、`/api/datafiles/`（列表/下载，分页）、`/api/datasets/`（加目标文件夹/命名/保留）。
- 门户：`/api/portal/tree/`（可见文件夹树）、`/api/portal/folders/{id}/files/`、`/api/portal/search/?q=`、`/api/portal/files/{id}/download/`（均按可见性）。
- 移除：`/api/categories/`、`/api/grants/`、`/api/executions/`、旧 portal/datasets 接口。
- 数据库按工程约定**清空重建迁移**。

## 三、验证（三维度审核）

- **维度一·功能正确性**：`pytest` **70 例全过**（文件夹可见性递归/部门/个人、文件夹 CRUD 防环、授权二选一、run→新增命名文件、保留清理、门户树/文件/搜索/下载鉴权、用户/成员、审计）；`check`/`ruff`/`black` 全清。
- **维度二·健壮性/扩展性**：文件夹树支持任意层级 + 防环；授权递归；命名/保留可配；可见性集中一处。
- **维度三·关联影响**：清库重建；**前端暂与新接口不匹配（v0.18/v0.19 重做）**，这是中间态，仅在 dev。

## 四、下一版本

- **v0.18 管理端前端**：文件夹管理树（建/重命名/移动/删）+ 数据集编辑（选目标文件夹/命名/定时/保留）+ 文件夹授权（部门/个人）+ 部门与用户。
- **v0.19 用户端**：生产级**文件浏览器**（文件夹树 + 文件列表/大小/行数/时间/搜索/排序/下载/面包屑）。
