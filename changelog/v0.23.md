# v0.23 版本变更说明

> 发布日期：2026-06-11　|　分支：**dev**　|　负责人：henry + AI
> 主题：**管理员效率 —— 批量操作 / 运行健康看板 / 失败友好展示**

迭代计划第 2 版。94 项测试通过（v0.22 的 89 + 新增 5），ruff 干净，前端 build 通过。

## 一、本次变更

### 批量操作（feat）
- **批量运行**：数据集列表多选 → 顶部"批量运行"，后端 `POST /api/datasets/batch-run/` 逐个异步触发。
- **批量改保留**：多选 → "批量改保留"对话框，统一设置 keep_count/keep_days，
  后端 `POST /api/datasets/batch-retention/`。

### 运行健康看板（feat）
- DataFile 新增 `duration_ms` 字段（迁移 `0002`），运行成败都记录耗时。
- 新接口 `GET /api/datasets/run-health/?days=N`：近 N 天总数/成功率/平均耗时/失败数/运行中、
  最近 10 条失败、最慢 5 个数据集。
- 管理首页新增"运行健康 · 近 7 天"面板：成功率/平均耗时/失败/运行中 四指标 + 最近失败列表。

### 失败友好展示（feat）
- 数据集列表"失败"标签可点击，弹出该次运行的完整 `error_msg`。
- 新增"运行中"黄色标签（配合 v0.22 异步运行）。

## 二、改动文件
- 后端：`apps/execution/models.py`(+duration_ms)、`apps/execution/migrations/0002_*`、

- 前端：`api/dataset.ts`(批量+看板接口)、`views/admin/DatasetView.vue`(多选/批量/失败详情)、
  `views/admin/AdminHome.vue`(健康看板)。
- 测试：`tests/test_v023_admin.py`(新，5 项)。
