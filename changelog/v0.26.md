# v0.26 版本变更说明

> 发布日期：2026-06-11　|　分支：**dev**　|　负责人：henry + AI
> 主题：**参数化查询 + 图表数据**

迭代计划第 5 版。116 项测试通过（v0.25 的 108 + 新增 8），ruff 干净，前端 build 通过。

## 一、本次变更

### 参数化查询（feat，后端完整 + API）
- 复用 Dataset 已有 `params` 字段（形如 `[{"name":"dt","label":"日期","default":"2026-01-01"}]`）。
- `services.resolve_params(dataset, provided)`：合并定义默认值与本次传入值，**只接受已定义的参数名**
  （丢弃未定义键，防止注入额外绑定变量）。
- 参数作为**绑定变量**贯通 preview / run / 异步 / 调度全链路（SQL 用各自数据库原生占位符，
  Oracle `:name` / MySQL `%(name)s`，由绑定变量传入，天然防注入）。
- `POST /api/datasets/{id}/run/` 与 `preview/` 支持 body `params`；定时运行用参数默认值。

### 图表数据（feat，后端 + API）
- `GET /api/datasets/{id}/chart-data/?x=列&y=列&agg=sum|count|avg`：
  读最新成功文件，按 x 列分组对 y 列聚合，返回 `{labels, values}`，供前端渲染柱/饼图。

## 二、改动文件

- 前端：`api/dataset.ts`(runDataset 带 params + getChartData)。
