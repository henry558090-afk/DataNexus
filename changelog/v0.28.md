# v0.28 版本变更说明

> 发布日期：2026-06-11　|　分支：**dev**　|　负责人：henry + AI
> 主题：**新功能上线后的缺陷排查与修复 + 审批闭环 + 前端可视化入口**

继 v0.23–v0.27 加完功能后，站在技术总监视角对**新代码**做二次排查，修掉 6 类会在生产出问题的缺陷；
并补全审批闭环与所有前端可视化入口。135 项测试通过，ruff 干净，前端 build 通过。

## 一、缺陷修复（本版重点）

### 🔴 高
- **B1 临时文件磁盘泄漏**：选列/脱敏下载原先用 `NamedTemporaryFile(delete=False)` 且从不清理，
  每次下载在 temp 留一个永久 .xlsx → 磁盘被慢慢撑满。
  → 改为 `masking.stream_filtered_xlsx`：openpyxl read_only+write_only **流式写入内存 BytesIO**，
  不落临时文件。
- **B2 异步运行线程无上限**：每次运行 `threading.Thread().start()`，批量运行会瞬间起几十上百个线程、
  每个开一条业务库连接 → 线程/连接耗尽。
  → 改用**有界线程池** `ThreadPoolExecutor(max_workers=DATASET_RUN_CONCURRENCY)`（默认 4），超出排队。
- **B3 大文件读进 Web 进程内存**：图表数据 / 脱敏下载原先全量读（最多 10 万行）进同步请求内存。
  → 脱敏下载改流式（同 B1）；图表读取加 `CHART_MAX_ROWS`（默认 5 万）上限，并返回 `truncated` 标记。

### 🟠 中
- **B4 参数占位符未校验**：SQL 用了 `:dt` 但参数定义没声明，过去要等运行才报错。
  → 保存数据集时校验（去注释/字符串后扫 `:name` 与 `%(name)s`）：未声明即 400，报哪个参数缺。
- **B5 企微 SSO token 进 URL**：回调原先 `redirect(?token=xxx)`，token 会进访问日志/浏览器历史/Referer。
  → 改**一次性短码**：回调下发 `?wecom_code=`，前端 POST `/auth/wecom/exchange/` 换 token，
    码 120s 有效、用完即删（DB 存储，兼容多 worker）。
- **B6 图表非数值静默当 0**：Y 列是 `"1,234"` 这类文本会全部变 0，图看着对其实错。
  → 改为跳过并在响应里返回 `skipped`（无法解析的行数）。

## 二、审批闭环 + 前端可视化入口（随本版合入）
- **审批闭环**：Folder 加 `requestable`（默认 False，敏感目录不露名）；门户「申请访问其他目录」入口；
  管理端目录管理「可被申请」开关；通过即自动授权。
- **前端入口**（应用 impeccable 设计系统，浅色 Restrained）：企业微信登录、审批中心、审计可视化图表、
  门户收藏/更新提示/最近下载、数据集设置（参数/推送/脱敏/图表）、参数运行表单。
- **修复**：登出需点两次（异步登出导致路由先于清 token 执行）→ 改同步先清本地、后台失效服务端 token。
