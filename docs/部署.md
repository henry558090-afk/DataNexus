# data-nexus Linux 部署指南（完整版）

> **不需要 Docker、不需要 Nginx、不需要 PostgreSQL。**
> 生产 = 同一台 Linux 上两个进程：① `gunicorn`（一个进程同时发**前端页面**和 **/api**）② `runscheduler`（定时跑数）。
> 元数据库默认 **SQLite**（一个文件），也可改用你现有的 **MySQL**。

---

## 0. 架构总览

```
          浏览器
            │  http(s)
            ▼
   ┌──────────────────────────┐
   │ gunicorn (Django)        │  ← deploy/start.sh 启动
   │  · WhiteNoise 发前端 dist │
   │  · /api/* 后端接口        │
   └───────┬──────────────────┘
           │
   ┌───────┴───────┐   ┌────────────────────────┐
   │ SQLite 文件   │   │ runscheduler (定时进程) │ ← 同一脚本启动
   │ 或 现有 MySQL │   │  按数据集定时跑 SQL     │
   └───────────────┘   └───────────┬────────────┘
                                   │ 只读连接
                       ┌───────────┴───────────┐
                       │  你的 Oracle / MySQL  │  ← 外部业务库（只读账号）
                       └───────────────────────┘
```

---

## ⭐ 你的专属路径（CentOS/麒麟 + SQLite + 域名 HTTPS）

> 你已确认：CentOS/RHEL/麒麟、元数据库用 SQLite、服务器上构建前端、域名 + HTTPS（走你现有网关）。照这条走即可，下面各节是通用细节。

```bash
# 1) 装环境（CentOS/RHEL/麒麟）
sudo yum install -y python3 python3-pip git
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 2) 拉代码 + 一键初始化（自动建 venv/装依赖/构建前端/生成.env/迁移）
sudo mkdir -p /opt && cd /opt
git clone <你的仓库地址> data-nexus && cd data-nexus
bash deploy/setup.sh            # ← 记下它打印的 FERNET_KEY 并备份！

# 3) 改 .env 两行（域名 HTTPS 必须）
#    ALLOWED_HOSTS=data.corp.com
#    CSRF_TRUSTED_ORIGINS=https://data.corp.com
vi backend/.env

# 4) 建管理员 + 启动
backend/.venv/bin/python backend/manage.py createsuperuser
bash deploy/start.sh           # 监听 0.0.0.0:8000
```

**最后一步交给你的网关/运维**：让现有反向代理把 `https://data.corp.com` 终止 TLS 后转发到 `http://本机IP:8000`，并带上 `X-Forwarded-Proto: https` 头（平台据此识别 HTTPS）。本平台**不需要你装 Nginx**。

> 参考：若你们网关恰好是 Nginx，转发段大致是
> ```nginx
> location / {
>     proxy_pass http://127.0.0.1:8000;
>     proxy_set_header Host $host;
>     proxy_set_header X-Forwarded-Proto $scheme;   # 关键：让 Django 知道是 https
>     proxy_set_header X-Real-IP $remote_addr;
> }
> ```

## 1. 服务器环境要求（你需要准备的）

| 项 | 要求 | 说明 |
|---|---|---|
| 操作系统 | Linux（Ubuntu 20.04+/Debian 11+ 或 CentOS 7+/RHEL/麒麟等） | 64 位 |
| Python | **3.11 ~ 3.13** | `python3 --version` |
| Node.js | **18+**（仅构建前端用） | 也可在本地构建后上传 `frontend/dist`，服务器免装 Node |
| Git | 任意 | 拉代码用 |
| 端口 | 默认 **8000**（可改） | 确保未被占用、防火墙放行 |
| 业务数据库 | Oracle / MySQL 的**只读账号** | 平台只读连它取数；Oracle 用 thin 模式，**无需装 Instant Client** |
| 磁盘 | 视生成文件量而定 | Excel 存在 `backend/media/` |

> 不需要安装 Oracle/MySQL 客户端库：Oracle 走 `oracledb` thin 纯 Python，MySQL 走 `PyMySQL` 纯 Python。

### 1.1 安装系统依赖

**Ubuntu / Debian：**
```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git
# 需要在服务器构建前端时再装 Node：
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

**CentOS / RHEL：**
```bash
sudo yum install -y python3 python3-pip git
# Node：
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
```

## 2. 部署步骤（三步）

### 第 1 步：拉代码
```bash
sudo mkdir -p /opt && cd /opt
git clone <你的仓库地址> data-nexus
cd data-nexus
```

### 第 2 步：一次性初始化（自动）
```bash
bash deploy/setup.sh
```
这个脚本会自动完成：
1. 建 Python 虚拟环境 `backend/.venv`
2. 安装后端依赖（gunicorn / Django / oracledb / PyMySQL / whitenoise …）
3. 构建前端（生成 `frontend/dist`）
4. **生成 `backend/.env` 并自动填好 `SECRET_KEY` 和 `FERNET_KEY`**
5. 数据库迁移 + 收集静态资源

> ⚠️ 脚本会打印一串 **FERNET_KEY**，请**立刻单独备份**（它是解密数据源密码的钥匙，丢了已存的数据源密码就全废了）。

> 若服务器没装 Node：先在本地 `cd frontend && npm run build`，把生成的 `frontend/dist` 整个目录上传到服务器同位置，再跑 `setup.sh`（它检测到没 npm 会跳过构建）。

### 第 3 步：建管理员 + 启动
```bash
# 建超级管理员（按提示输账号密码）
backend/.venv/bin/python backend/manage.py createsuperuser

# 一键启动（web + 定时）
bash deploy/start.sh
```
看到 `✅ 已启动` 即成功。浏览器打开 `http://服务器IP:8000/`。

## 3. 配置文件 `backend/.env` 速查

`setup.sh` 已自动生成，**通常只需改 `ALLOWED_HOSTS`**：

| 变量 | 作用 | 生产建议 |
|---|---|---|
| `SECRET_KEY` | Django 签名密钥 | 已自动生成，勿外泄 |
| `DEBUG` | 调试开关 | **保持 `False`** |
| `ALLOWED_HOSTS` | 允许访问的域名/IP | 改成你的，如 `data.corp.com,10.0.0.8`（逗号分隔；内网图省事可暂用 `*`）|
| `CORS_ALLOWED_ORIGINS` | 跨域来源 | **留空**（前后端同源，无需跨域）|
| `CSRF_TRUSTED_ORIGINS` | 受信任的 HTTPS 来源 | **域名 HTTPS 必填**，如 `https://data.corp.com`（不填后台登录会 CSRF 403）|
| `FERNET_KEY` | 数据源密码加密钥匙 | 已自动生成，**务必备份** |
| `QUERY_MAX_ROWS` | 单次取数行数上限 | 默认 10 万 |
| `QUERY_TIMEOUT_SECONDS` | 取数超时秒 | 默认 60 |
| `EXECUTION_KEEP` | 每数据集保留文件数 | 默认 20 |
| `DATABASE_URL` | 平台元数据库 | 不填=用 SQLite；要用现有 MySQL 见下 |

**改完 `.env` 后重启：** `bash deploy/start.sh`（脚本会先停再起）。

### 3.1 （可选）元数据库改用现有 MySQL
在你的 MySQL 建一个空库（如 `datanexus`），编辑 `backend/.env`：
```ini
DATABASE_URL=mysql://用户名:密码@主机:3306/datanexus
```
然后重新 `bash deploy/start.sh`（会自动迁移到 MySQL）。不填则用 SQLite，零配置、内网小规模够用。

## 5. 开机自启 + 崩溃自动拉起（推荐，systemd）

`start.sh` 用 `nohup` 起进程，**重启服务器后不会自动恢复**。生产建议用 systemd 托管：

`/etc/systemd/system/datanexus-web.service`
```ini
[Unit]
Description=data-nexus web
After=network.target

[Service]
User=root
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 3
Restart=always

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/datanexus-scheduler.service`
```ini
[Unit]
Description=data-nexus scheduler
After=network.target

[Service]
User=root
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/python manage.py runscheduler
Restart=always

[Install]
WantedBy=multi-user.target
```

启用：
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now datanexus-web datanexus-scheduler
sudo systemctl status datanexus-web      # 查看状态
```
> 用 systemd 后就不用 `start.sh` 了；改完代码/配置 `sudo systemctl restart datanexus-web datanexus-scheduler`。
> ⚠️ 定时进程**只能起一个实例**：用了 systemd 就别再跑 `start.sh`，避免重复。

## 6. （可选）HTTPS / 域名

平台本身不强制 HTTPS。若需要，**不必为它单独装 Nginx**——挂到你公司**已有的反向代理 / 负载均衡 / 网关**上，由它终止 TLS 并转发到 `http://本机:8000` 即可。三个要点：
1. `.env` 的 `ALLOWED_HOSTS` 加上域名（如 `data.corp.com`）。
2. `.env` 的 `CSRF_TRUSTED_ORIGINS` 加上 `https://域名`（否则后台 `/django-admin/` 登录会 CSRF 403）。
3. 反代要透传 `X-Forwarded-Proto: https` 头（平台据此识别 HTTPS、启用安全 Cookie）。改完 `.env` 重启平台。

## 9. 常见问题

- **端口被占用**：`DN_PORT=9000 bash deploy/start.sh`，或 `ss -ltnp | grep 8000` 找占用进程。
- **页面打开 400 / DisallowedHost**：把你的域名/IP 加进 `.env` 的 `ALLOWED_HOSTS`，重启。
- **`bad interpreter` 报错**：脚本被改成了 CRLF 换行。执行 `sed -i 's/\r$//' deploy/*.sh` 后重试。
- **连不上 Oracle/MySQL 业务库**：确认服务器到数据库网络通、只读账号密码对、防火墙放行；Oracle 用 thin 模式无需客户端。
- **数据源密码解不开**：`FERNET_KEY` 变了或丢了。恢复原 KEY，或在“数据源”页重新填一次密码。
- **静态/前端 404**：确认 `frontend/dist` 存在（跑过 `npm run build` 或上传过 dist），并执行过 `collectstatic`（`start.sh` 会自动做）。
