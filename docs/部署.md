# data-nexus 部署文档（Linux · 最小方案）

> **不需要 Docker、不需要 Nginx、不需要 PostgreSQL。** 只要：Python + 两个进程 + 一个 SQLite 文件。
> Django（gunicorn）一个进程同时发**前端页面**和**API**；定时调度单独一个进程。

---

## 0. 架构（就这么简单）

```
浏览器 ──► gunicorn(Django)  ① 发前端 SPA（WhiteNoise）  ② 发 /api/*
                │
                ├─ SQLite 文件（平台元数据；可选改 MySQL）
                └─ media/ 目录（生成的 Excel）

另一个进程：python manage.py runscheduler   ② 定时跑数据集
外部：你的 Oracle / MySQL 业务库（只读账号）
```

> 业务数据源（Oracle/MySQL）是**外部已有的库**，平台只读连它。平台自身元数据默认用 SQLite。

---

## 1. 前置

- Linux 服务器，Python 3.11+，Node 18+（仅构建前端用，可在本地构建后上传 dist）。
- 代码：`git clone` 本仓库。

## 2. 后端

```bash
cd backend
python -m venv .venv
. .venv/bin/activate
pip install -r requirements-prod.txt        # 含 gunicorn
```

## 3. 前端（构建一次，产物交给 Django 发）

```bash
cd ../frontend
npm ci
npm run build                                # 生成 frontend/dist
# 之后 Django 会自动发 dist（WhiteNoise），无需 Nginx
```

> 也可在本地 `npm run build` 后，把 `frontend/dist` 上传到服务器同位置。

## 4. 配置 `.env`（backend/.env）

```ini
SECRET_KEY=<python -c "import secrets;print(secrets.token_urlsafe(50))">
DEBUG=False
ALLOWED_HOSTS=your.domain.com,10.0.0.5
CORS_ALLOWED_ORIGINS=https://your.domain.com

# ⚠️ 必须生成一次并【备份】，一旦丢失/变更，已存数据源密码全部解不开！
FERNET_KEY=<python -c "from cryptography.fernet import Fernet;print(Fernet.generate_key().decode())">

# 元数据库：默认 SQLite（不用配）。如要用现有 MySQL：
# DATABASE_URL=mysql://user:password@host:3306/datanexus

EXECUTION_KEEP=20
QUERY_MAX_ROWS=100000
QUERY_TIMEOUT_SECONDS=60
```

## 5. 初始化

```bash
cd ../backend
. .venv/bin/activate
python manage.py migrate
python manage.py collectstatic --noinput     # 收集 admin/DRF 静态，WhiteNoise 发 /static/
python manage.py createsuperuser              # 建超级管理员
```

## 6. 起两个进程

**Web（gunicorn）：**
```bash
gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 3
```

**定时调度（单独进程，只能起一个实例）：**
```bash
python manage.py runscheduler
```

打开 `http://服务器:8000/` 即可（前端 + API 同一端口）。Django 后台在 `/django-admin/`。

---

## 7. 用 systemd 守护（推荐，开机自启 + 崩溃重启）

`/etc/systemd/system/datanexus-web.service`：
```ini
[Unit]
Description=data-nexus web
After=network.target

[Service]
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 3
Restart=always

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/datanexus-scheduler.service`：
```ini
[Unit]
Description=data-nexus scheduler
After=network.target

[Service]
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/python manage.py runscheduler
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now datanexus-web datanexus-scheduler
```

---

## 8. HTTPS（可选）

平台本身不强制 HTTPS。若需要，**不必为它单独装 Nginx**——挂到你公司**现有的反向代理/负载均衡**上，由它终止 TLS 转发到 `:8000` 即可。

---

## 9. 升级流程

```bash
git pull
cd frontend && npm ci && npm run build
cd ../backend && . .venv/bin/activate
pip install -r requirements-prod.txt
python manage.py migrate
python manage.py collectstatic --noinput
sudo systemctl restart datanexus-web datanexus-scheduler
```

---

## 10. 备份

- `backend/db.sqlite3`（或你的 MySQL）：平台元数据。
- `backend/media/`：生成的 Excel（也可接受丢失，重跑即可）。
- **`FERNET_KEY`：务必单独安全备份**（解数据源密码的钥匙）。
