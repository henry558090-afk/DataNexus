# data-nexus 部署运维手册（给运维）

> **照这一份做就够，不用看别的文档。**
> 平台 = **一台 Linux 上的两个进程**：① `gunicorn` 同时发**前端页面**和 **/api 接口**；② `runscheduler` 定时跑数。
> **不需要 Docker、不需要 Nginx、不需要单独装数据库。**

```
        浏览器  http(s)
           │
           ▼
   ┌──────────────────────────┐
   │ gunicorn (一个进程)       │  ← deploy/start.sh 启动，监听 47821
   │  · 发前端页面 dist        │
   │  · /api/* 后端接口        │
   └───────┬──────────────────┘
           │
   ┌───────┴───────┐   ┌────────────────────────┐
   │ SQLite 文件   │   │ runscheduler (定时进程) │ ← 同一脚本启动
   │ (平台元数据)  │   │  按数据集定时跑 SQL     │
   └───────────────┘   └───────────┬────────────┘
                                   │ 只读连接
                       ┌───────────┴───────────┐
                       │  业务 Oracle / MySQL  │  ← 外部库，用只读账号
                       └───────────────────────┘
```

---

## 一、对外端口（重点先说）

- **本平台只对外暴露 1 个端口：`47821`（冷门端口，已设为默认）。**
- 前端不是独立服务，和 `/api` 由**同一个进程**一起发出，所以**只有这一个端口**，不存在"前端端口 / 后端端口"两个。
- 请在**防火墙 / 安全组放行 `47821/tcp`**（仅对需要访问平台的网段开放即可）。

> 若 47821 被占用或不符合你们端口规范，告诉我换一个，或临时用
> `DN_PORT=你的端口 bash deploy/start.sh` 启动。

## 二、系统需要什么

### 2.1 服务器软件（必须装）

| 软件 | 用途 | CentOS/RHEL/麒麟 安装 | 检查 |
|---|---|---|---|
| **git** | 从代码库拉代码 | `sudo yum install -y git` | `git --version` |
| **Python 3.11~3.13** + pip | 运行后端 | `sudo yum install -y python3 python3-pip` | `python3 --version` |
| **Node.js 18+** + npm | 构建前端（可选*） | `curl -fsSL https://rpm.nodesource.com/setup_20.x \| sudo bash -` 再 `sudo yum install -y nodejs` | `node -v` |

Ubuntu/Debian 改用 `apt`：`sudo apt install -y python3 python3-venv python3-pip git`，Node 同理用 `deb.nodesource.com`。

> *若服务器不能装 Node：我会在本地构建好 `frontend/dist` 一起给你，你跳过 Node 即可。

### 2.2 不需要装（澄清）

- ❌ Docker、❌ Nginx、❌ PostgreSQL
- ❌ Oracle 客户端 / Instant Client（用 `oracledb` thin 纯 Python 连）
- ❌ MySQL 客户端库（用 `PyMySQL` 纯 Python 连）

### 2.3 服务器已自带（无需装）

- **sshd**（登录服务器 / 传代码用，Linux 默认就有）
- **systemd**（托管进程、开机自启，见第四节）

### 2.4 网络要开通（找网络/DBA 配合）

1. **服务器 → 公司内网 GitLab**：拉代码用（地址我提供）。
2. **服务器 → 业务数据库（Oracle / MySQL）**：用**只读账号**取数。请确认服务器到库网络通、防火墙放行端口；Oracle 无需装客户端。
3. **用户浏览器 → 服务器 `47821`**（或经你们网关的 HTTPS 域名）。

### 2.5 磁盘

视生成文件量而定，生成的 Excel 存在 `backend/media/`。

## 三、首次部署（4 步，约 10 分钟）

```bash
# 1) 拉代码（地址用我给你的公司内网 GitLab 地址）
sudo mkdir -p /opt && cd /opt
git clone <公司内网 GitLab 地址> data-nexus && cd data-nexus

# 2) 一键初始化（自动建 venv、装依赖、构建前端、生成配置、迁移）
bash deploy/setup.sh
#    ⚠️ 这一步会打印一串 FERNET_KEY，请【复制保存】，第六节要回给我！

# 3) 建管理员账号（按提示输用户名/密码，记下来给我）
backend/.venv/bin/python backend/manage.py createsuperuser

# 4) 启动（监听 0.0.0.0:47821）
bash deploy/start.sh
```

看到 `✅ 已启动` 即成功。浏览器访问 `http://服务器IP:47821/` 能打开登录页就对了。

**自检：**
```bash
curl http://127.0.0.1:47821/api/health/      # 返回 200 即正常
```

> 走域名 HTTPS 的话，部署后编辑 `backend/.env`，把 `ALLOWED_HOSTS` 设成域名、`CSRF_TRUSTED_ORIGINS` 设成 `https://域名`，再重启（详见第八节）。

## 四、配置开机自启（强烈建议）

`start.sh` 用 nohup 启动，**服务器重启后不会自动恢复**。生产请用 systemd 托管。新建下面两个文件（路径 `/opt/data-nexus` 按实际调整）：

`/etc/systemd/system/datanexus-web.service`
```ini
[Unit]
Description=data-nexus web
After=network.target

[Service]
User=root
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/gunicorn config.wsgi:application --bind 0.0.0.0:47821 --workers 3
Restart=always

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/datanexus-scheduler.service`
```ini
[Unit]
Description=data-nexus scheduler
After=network.target

[Service]
User=root
WorkingDirectory=/opt/data-nexus/backend
EnvironmentFile=/opt/data-nexus/backend/.env
ExecStart=/opt/data-nexus/backend/.venv/bin/python manage.py runscheduler
Restart=always

[Install]
WantedBy=multi-user.target
```

启用：
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now datanexus-web datanexus-scheduler
sudo systemctl status datanexus-web      # 看是否 running
```

> ⚠️ 用了 systemd 就**别再手动跑 `start.sh`**，避免定时进程起两份重复跑数。

## 五、后续提交代码 → 更新到服务器（日常发版）

代码库是中转站：开发者推到**公司内网 GitLab**，服务器从 GitLab 拉。

```
   开发者本机              公司内网 GitLab            服务器(生产)
  ┌──────────┐ git push  ┌──────────────┐ git pull ┌──────────────┐
  │ 改代码    │ ────────► │   代码仓库    │ ───────► │ 拉最新 + 重启 │
  └──────────┘           └──────────────┘          └──────────────┘
```

**服务器侧操作（每次更新只需这几条）：**
```bash
cd /opt/data-nexus
git pull
bash deploy/setup.sh      # 依赖或前端有变化时执行；只改后端 .py 可跳过
# 重启：
sudo systemctl restart datanexus-web datanexus-scheduler   # 用了 systemd
# 或（没用 systemd 时）：
bash deploy/start.sh
```

> **谁做这步？** 两种分工，二选一：
> - **运维负责**：开发者 push 后通知你，你上服务器 `git pull` + 重启。
> - **开发者自己负责**：开发者 ssh 上来自己 pull + 重启，你只在最初配好环境和权限。

**`git pull` 不会覆盖服务器本地数据** —— 以下文件已被 gitignore，是服务器独有的，拉代码不动它们：
`backend/.env`、`backend/db.sqlite3`、`backend/media/`、`node_modules`、`frontend/dist`。

> 拉到带新数据库迁移的版本后，`setup.sh` / `start.sh` 会自动 `migrate`，不用手动。

## 八、配置文件 `backend/.env` 速查

`setup.sh` 已自动生成，**通常只需改 `ALLOWED_HOSTS`**（走域名 HTTPS 时再加 `CSRF_TRUSTED_ORIGINS`）。

| 变量 | 作用 | 建议 |
|---|---|---|
| `SECRET_KEY` | Django 签名密钥 | 已自动生成，勿外泄 |
| `DEBUG` | 调试开关 | **保持 `False`** |
| `ALLOWED_HOSTS` | 允许访问的域名/IP | 改成你的，如 `data.corp.com,10.0.0.8`（逗号分隔；内网图省事可暂用 `*`）|
| `CSRF_TRUSTED_ORIGINS` | 受信任 HTTPS 来源 | **域名 HTTPS 必填**，如 `https://data.corp.com`（不填后台登录会 CSRF 403）|
| `FERNET_KEY` | 数据源密码加密钥匙 | 已自动生成，**务必备份**（见第六节）|
| `QUERY_MAX_ROWS` | 单次取数行数上限 | 默认 10 万 |
| `QUERY_TIMEOUT_SECONDS` | 取数超时秒 | 默认 60 |
| `EXECUTION_KEEP` | 每数据集保留文件数 | 默认 20 |
| `DATABASE_URL` | 平台元数据库 | 不填=用 SQLite；要用现有 MySQL 见 8.1 |

改完 `.env` 后重启：`bash deploy/start.sh`（或 systemd restart）。

### 8.1（可选）元数据库改用现有 MySQL

在你的 MySQL 建一个空库（如 `datanexus`），编辑 `backend/.env`：
```ini
DATABASE_URL=mysql://用户名:密码@主机:3306/datanexus
```
然后重启（会自动迁移到 MySQL）。不填则用 SQLite，零配置、内网小规模够用。

### 8.2（可选）HTTPS / 域名

平台本身不强制 HTTPS。若需要，**不必单独装 Nginx**——挂到你公司**已有的反向代理 / 网关**上，由它终止 TLS 并转发到 `http://本机:47821`。三个要点：
1. `.env` 的 `ALLOWED_HOSTS` 加上域名。
2. `.env` 的 `CSRF_TRUSTED_ORIGINS` 加上 `https://域名`。
3. 反代透传 `X-Forwarded-Proto: https` 头。改完重启。

> 若你们网关是 Nginx，转发段大致：
> ```nginx
> location / {
>     proxy_pass http://127.0.0.1:47821;
>     proxy_set_header Host $host;
>     proxy_set_header X-Forwarded-Proto $scheme;   # 关键
>     proxy_set_header X-Real-IP $remote_addr;
> }
> ```

## 十、出问题先看这里

- **页面打不开 / 400 DisallowedHost** → 把服务器 IP/域名加进 `backend/.env` 的 `ALLOWED_HOSTS`，重启。
- **端口被占用** → `ss -ltnp | grep 47821` 查占用，或换端口 `DN_PORT=39517 bash deploy/start.sh`。
- **`bad interpreter` 报错** → 脚本被改成了 Windows 换行：`sed -i 's/\r$//' deploy/*.sh` 后重试。
- **连不上业务库** → 确认网络通、只读账号密码对、防火墙放行；Oracle 用 thin 模式无需客户端。
- **数据源密码解不开** → `FERNET_KEY` 变了或丢了。恢复原 KEY，或在"数据源"页重新填密码。
- **静态/前端 404** → 确认 `frontend/dist` 存在、`setup.sh` 跑过（它会自动 `collectstatic`）。
